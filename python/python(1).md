输入完成后，不会有任何提示，Python交互式命令行又回到>>>状态了。那我们刚才输入的内容到哪去了？答案是存放到name变量里了。可以直接输入name查看变量内容：
什么是变量
你可以把变量想象成一个箱子，你只能往里放一个东西，如果你想要放其他东西你就得把以前的拿出来
变量名就是箱子，变量就是你想要放的东西，你只能放一个这就说明不能有俩个相同的变量，新的变量会替换旧的变量
那你也许就会有疑问如果不能有相同的变量那为什么不直接写在代码里，这叫做硬编码

“硬编码”（Hardcoding）指的是**把具体的数据直接死死地写在代码里**（比如直接写 `print("小明")`、`score = 100`），而不是用变量或 `input()` 来动态获取。

不鼓励硬编码，主要有以下三个原因：

---

1. 缺乏灵活性：程序会变成“死”的

想象一下，你写了一个登录系统，如果用**硬编码**：

```python
# 硬编码：把名字死死写在代码里
print("欢迎你，小明！")

```

这个程序就变成了“小明专属版”。如果小华、小红来登录，系统依然会说“欢迎你，小明！”。

而如果使用**变量和输入（动态获取）**：

```python
name = input("请输入你的名字：")
print("欢迎你，" + name + "！")

```

现在，无论谁来用这个程序，它都能正确地打招呼。**不硬编码，程序才是“活”的。**

---

2. 维护成本极高：“牵一发而动全身”

假设你正在开发一款游戏，里面的“怪物初始血量”是 `500`。你在代码里的 100 个不同的地方（比如计算伤害、画血条、判断胜负）都直接写了 `500`。

过了一个月，老板说：“游戏太难了，把怪物血量改成 `300` 吧。”

* **硬编码的下场**：你必须在成千上万行代码里，眼花缭乱地去寻找这 100 个 `500`，手动一个一个改成 `300`。一旦漏掉一个，程序就会出 Bug。
* **使用变量的做法**：
```python
monster_hp = 500  # 只在开头定义一次

```


你只需要把第一行的 `500` 改成 `300`，后面所有引用 `monster_hp` 的地方就自动全变了！

---

3. 安全隐患：容易泄露核心秘密

如果你的代码里需要用到银行卡密码、服务器密钥或者你的家庭住址。

* **硬编码**：直接把密码写在代码里 `password = "123456"`。一旦你把代码分享给别人，或者上传到网络上，你的秘密就彻底曝光了。
* **非硬编码**：代码里只写 `password = input("请输入密码")` 或者从一个加密的隐私文件里去读取。代码本身不包含任何敏感信息，非常安全。

---

什么时候能用硬编码？

硬编码也不是绝对不能用。如果你确定某个东西**永远绝对不会变**（比如一年有 12 个月、圆周率 $\pi$ 的近似值 $3.14$），可以直接写死。

但只要这个数据**未来可能会变**，或者**需要根据不同用户发生变化**，就应该用变量或输入来代替它。

print不仅只能打印字串符还可以打印变量
```python
print(name)
```
